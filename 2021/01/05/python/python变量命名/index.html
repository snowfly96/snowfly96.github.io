python代码规范、变量命名与注释

发表于2021-01-05|更新于2021-01-05|python
|阅读量:

代码规范

1. 编码

一般情况下，文件采用UTF-编码，文件头加上# -*-coding:utf8 -*-标识

2.引号

  • 自然语言采用双引号"..."
  • 机器标识采用单引号..., 比如dic中的key
  • 正则表达式，使用双引号r"..."
  • 字符串文档采用三个双引号"""..."""

3. 空行

  • 模块级别函数之间空两行
  • 类成员函数之间空一行

变量命名

1. 模块

模块尽量采用小写命名，首字母为小写，尽量不采用下划线（除非多个单词）

2. 函数

函数推荐采用小写加下划线

3. 类名

类名采用(CamelCase)驼峰命名风格，首字母大写，私有类别可以加一个下划线

相关的类可以放在一个模块中，不需要像java每一个类放在一个模块中

4. 变量

变量尽量采用小写，多个单词采用下划线隔开

常量采用大写，多个单词采用下划线

变量函数下划线

1. _var单下划线

该变量作为一个提示(hint), 表明这个一个类内部变量，但是外部类仍然可以访问到，一般情况下定义为单下划线方便类与子类访问

2. var_单下划线

解决命名冲突

3.__var双下划线

私有变量，无法在外部被使用，子类也无法访问

4. __var__双下划线

系统定义命名

5._func()

标准为私有函数，只是标注

6.__func()

标注为特殊函数，比如操作符号重载

7._单个下划线

表示一个不重要或者临时的变量
